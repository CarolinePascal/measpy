class measpy.measurement.Measurement(**params)

Bases: object

The class Measurement allows to simply define and perform a measurement.

Initialization parameters:

Parameters

  • fs (int) – Sampling frequency, defaults to 44100.

  • dur (float) – Duration in seconds, defaults to 2.0.

  • in_map (list(int)) – Map of the inputs used on the device, defaults to [1,2].

  • in_cal (list(str)) – Calibrations of the incoming signals, in volts/units, defaults to a list of 1.0.

  • in_unit (list(str)) – Measurement units, defaults to a list of ‘V’.

  • in_dbfs – Input voltage for a unitary value in the resulting signal, defaults to 1.0.

  • in_name (list(str)) – Short name of the measurement signals (used as keys for the data dictionnary), defaults to ‘In’+str(n)

  • in_desc – Description of the measurement signals, defaults to ‘Thi is input ‘+str(n)

  • out_sig (float) – Output signal type (‘noise’, ‘logsweep’, ‘file.wav’ or None) where file.wav is a wave file string.

  • extrat (tuple(int,int)) – (not really useful yet) Additionnal extra time before and after (additionnal time), in sample numbers.

  • out_map (list(int)) – Map of the outputs used on the device, defaults to [1]

  • out_amp (float) – Output amplitude, defaults to 1.0

  • out_dbfs – Output voltage for a unitary value in the sent signal, defaults to 1.0.

  • extrat – Additionnal extra time before and after (additionnal time), in sample numbers.

  • out_sig_freqs (tuple(float,float)) – Minimum and maximum frequencies for the generated output signals (used if out_sig is ‘noise’ or ‘logsweep’)

  • fades (tuple(int,int)) – Fades in/out at the begining and end of the output signal, defaults to (0,0)

  • io_sync (int) – (not implemented yet) Specifies if in/out synchronization is done, and which type, defaults to 0 (no synchronization).

  • device_type (str or None) – Type of defice ‘audio’, ‘ni’, or ‘’, or None, defaults to ‘’. It can be eventually reacualized when running the measurement.

  • in_device (str) – Input device, defaults to ‘’

  • out_device (str) – Output device, defaults to ‘’

class measpy.signal.Signal(**kwargs)

Bases: object

The class signal describes a sampled data, its sampling frequency, its unit, the calibration and dbfs used for data acquisition. Methods are provided to analyse and transform the data.

Parameters

  • raw (1D numpy array, optional) – raw data of the signal, defaults to array(None)

  • volts (1D numpy array, optional) – raw data of the signal

  • values (1D numpy array, optional) – raw data of the signal

  • desc (str, optional) – Description of the signal, defaults to ‘A signal’

  • fs (int, optional) – Sampling frequency, defaults to 1

  • unit (str, optional) – Unit of the signal given as a string that pint can understand, defaults to ‘1’

  • cal (float, optional) – calibration in V/unit, defaults to 1.0

  • dbfs (float, optional) – dbfs of the input data acquisition card, defaults to 1.0

A signal is a temporal series of values. The object has the following properties:

  • desc : The description of the signal (string)

  • unit : The physical unit (pint.Unit)

  • cal : The calibration (in V/unit)

  • dbfs : The input voltage for a raw value of 1

  • fs : The sampling frequency

  • _rawvalues : A numpy array of raw values

Setters and getters properties:

  • values (values expressed in unit, calibrations applied)

  • volts (only dbfs applied)

  • raw (same as _rawvalues)

  • length (data length)

  • dur (duration in seconds)

  • time (time array)

harmonic_disto(nh=4, freqs=(20, 20000), delay=None)

Compute the harmonic distorsion of an in/out system using the method proposed by Farina (2000).

The signal object (`self`) has to be the response of a system to a logarithmic sweep created with the `Signal.log_sweep` method.

Parameters

  • nh (int, optional) – number pf harmonics, including harmonic 0, which is the linear part of the response, defaults to 4

  • freqs (tuple, optional) – frequencies between which the output signal sweeps, defaults to [20,20000]

  • delay (float, optional) – the mean delay between output and input, defaults to None. If None, the delay is estimated by calculating the mean values of the group delay between the freauencies of the sweep

Returns

A dictionary of Spectral objects representing the different harmonics as function of the frequencyu

Return type

dict of measpy.Spectral

class measpy.signal.Spectral(**kwargs)

Bases: object

Class that holds a set of values as function of evenly spaced frequencies. Usualy contains tranfert functions, spectral densities, etc.

Frequencies are not stored. If needed they are constructed using sampling frequencies and length of the values array by calling the property freqs.

Parameters

  • fs (int, optional) – Sampling frequency, defaults to 1

  • desc (str, optional) – Description, defaults to ‘Spectral data’

  • unit (str, unyt.Unit, optional) – Spectral data unit

  • values (numpy.array, optional) – Values of the pectral data

  • full (bool, optionnal) – If true, the full spectrum is given, from 0 to fs, if false, only up to fs/2

  • norm (string, optionnal) – Type of normalization “backward”, “ortho” or “full”. See numpy.fft doc.

values and dur cannot be both specified. If dur is given, values are initialised at 0
